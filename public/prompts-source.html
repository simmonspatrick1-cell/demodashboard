How to Use This Guide

This guide provides AI Connector-optimized prompts for creating NetSuite demo data through Claude. These prompts follow best practices for tool calling, metadata validation, and producing realistic, interconnected demo scenarios.


GLOBAL INSTRUCTION BLOCK

Before taking any action, ask me any clarifying questions necessary to avoid incorrect assumptions.

Whenever creating, updating, or analyzing NetSuite data, always:

  1. Call ns_getRecordTypeMetadata first to understand the structure.

  2. Use the metadata response to build valid JSON for ns_createRecord or ns_updateRecord.

  3. Validate all required fields before executing any tool.

  4. Review your own output before finalizing; flag anything incomplete.

  5. If referencing related records, use internal IDs only.

  6. When creating interconnected demo data, maintain realistic relationships.

  7. Include clickable NetSuite hyperlinks for any record you create:

    Format: https://td3049589.app.netsuite.com/app/accounting/transactions/[recordtype].nl?id=[ID]&whence=

  8. If unsure, state assumptions clearly and ask for missing information.

Pro Tips for Effective Demo Prep

Best Practices

Always use metadata rst: Never create records without calling ns_getRecordTypeMetadata

Ask clarifying questions: Don't assume subsidiary, currency, or required eld values

Validate relationships: Ensure internal IDs link correctly between related records


Provide hyperlinks: Always return clickable NetSuite URLs for created records

Maintain data integrity: Create realistic, interconnected data that tells a story


Use proper staging: Create records in "different stages" to demonstrate workows

Include variety: Vary amounts, dates, statuses to create realistic datasets

Verify before nalizing: Review your own output for completeness and accuracy


Data Quality Checks

Ensure all required elds are populated


Validate calculated elds (totals, balances, etc.)

Check date sequences are logical


Verify currency consistency within transactions

Conrm subsidiary alignment across related records

Test that transformations work correctly (Quote SO Invoice)

  4. Validation notes conrming data quality
